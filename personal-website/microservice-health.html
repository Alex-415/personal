Overview

A production-ready microservice health monitoring system that simulates a network of interconnected services with realistic failure scenarios, latency tracking, and dependency management. This project demonstrates my expertise in distributed systems architecture, concurrent programming (Go/Python), real-time data streaming, and cloud deployment.

üéØ Why I Built This

To showcase production-level skills that telecom and cloud infrastructure companies look for:

  • Distributed Systems Design: Service mesh simulation with dependency graphs and failure propagation
  • Concurrent Programming: Go goroutines and Python asyncio for high-performance parallel processing
  • Real-time Streaming: WebSocket implementation for live metric updates
  • Cloud-Native Deployment: Dockerized microservices on Render.com with CI/CD-ready architecture
  • Observability Patterns: Metrics collection, health checks, and structured logging

Key Features

  • Concurrent Service Simulation: Async Python tasks simulate 5-10 microservices with configurable dependencies
  • Real-time Metrics: Track latency, error rates, success rates, and uptime per service
  • Failure Propagation: Simulate cascading failures across dependent services
  • Circuit Breaker Pattern: Implement resilience patterns for microservice communication
  • WebSocket Streaming: Real-time metric updates to frontend dashboard
  • Interactive Dashboard: Visual service graph with color-coded health status and metric charts
  • Production Architecture: Dockerized backend, structured logging, graceful shutdown

Technical Implementation

Backend (Python + FastAPI)

  • Async endpoints with asyncio for concurrent service simulation
  • Semaphore-based concurrency control for metric updates
  • REST API: GET /services, GET /metrics, POST /simulate-failure
  • WebSocket endpoint for real-time event streaming
  • Dependency graph traversal for failure propagation
  • JSON structured logging with timestamps
  • Environment-based configuration (service count, error rates, latency)
  • Docker container ready for Render.com deployment

Frontend (React + TypeScript)

  • Dynamic service graph with node visualization
  • Color-coded health indicators (green/yellow/red)
  • Real-time metric charts (latency, error rate, success rate)
  • Interactive service selection and failure triggering
  • WebSocket client for live updates
  • Strong TypeScript typing throughout
  • Modular component architecture

Architecture Highlights

  • Concurrency-Safe Metrics: Atomic operations with asyncio locks
  • Dependency Management: Graph-based service relationships
  • Error Simulation: Configurable failure probability per service
  • End-to-End Latency: Track request paths across multiple services
  • Health Monitoring: /health endpoint for platform monitoring
  • Graceful Shutdown: Proper task cancellation and cleanup

Tech Stack

Python 3.11 FastAPI asyncio WebSockets React TypeScript Docker Render.com Netlify

Use Cases

  • Testing monitoring and alerting systems
  • Demonstrating circuit breaker and resilience patterns
